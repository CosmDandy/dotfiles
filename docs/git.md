# Git из терминала

Шпаргалка по своей обвязке. Цель — чтобы не тянуться к GUI: всё, что нужно
каждый день, уже забиндено.

## Алиасы

| Алиас | Команда |
|---|---|
| `gs` | `git status` |
| `gss` | `git status --short` — компактно, для быстрого взгляда |
| `gd` | `git diff` |
| `gdiff` | `git diff --color-words` — дифф по словам, а не по строкам |
| `glog` | `git log --oneline --graph --decorate` — граф веток |
| `gblame` | `git blame -w` — `-w` игнорирует правки отступов |
| `gp` | `git pull` |
| `gc` | `git clone` |
| `lg` | lazygit с автоподбором темы под фон терминала |

`gdiff` стоит освоить отдельно: на правках внутри длинной строки обычный дифф
показывает всю строку изменённой, а `--color-words` — только сами слова.

`gblame -w` не даёт «автору переформатирования» стать автором всего файла.

## lazygit

`lg` — TUI поверх git. Тема (светлая или тёмная) подбирается автоматически по
фону терминала, форсируется через `LG_THEME=light` или `dark`.

Что в нём быстрее, чем в командной строке: постановка отдельных строк в индекс
(`Space` на строке в диффе), интерактивный rebase, разбор конфликтов, просмотр
stash. Основными ветками считаются `main`, `master`, `stage`, `develop` — от
них считается «сколько коммитов впереди».

## Что тебя остановит

В репозитории стоят два хука, и оба сработают молча-неожиданно, если о них не
знать.

**pre-commit — сканирование секретов.** Гоняет `gitleaks protect --staged`.
Найдёт токен в изменениях — коммит не пройдёт. Если `gitleaks` не установлен,
коммит тоже не пройдёт: раньше он в этом случае молча пропускал, и сканирование
незаметно исчезало.

Осознанно пропустить (например, ложное срабатывание):

    GITLEAKS_SKIP=1 git commit -m "..."

**pre-push — защита основных веток.** Не даёт запушить в `main` или `master`,
если ты находишься не в этой ветке. Считает пушем и удаление ветки — при
`git push origin --delete master` он тоже сработает. Обход — `--no-verify`.

Хуки подключены глобально через `core.hooksPath = ~/.git-hooks`, то есть
работают во всех репозиториях, включая сабмодули.

## Соглашения

Коммиты — conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, `ci:`,
`chore:`. Ветки — `feature/<имя>`, `fix/<имя>`, `hotfix/<имя>`.

Основная ветка dotfiles называется `main` (переименована с `master` 21.07.2026).
Рабочие репозитории на GitLab по-прежнему на `master`/`develop` — поэтому в
конфиге lazygit перечислены обе.

## Обновить дотфайлы

    dfu

Делает `fetch` и `reset --hard` на upstream. **Стирает незакоммиченное** — это
и есть смысл («принудительно из облака»), но перед запуском стоит глянуть `gs`.

## Минимальный набор без алиасов

Если окажешься на чужой машине:

    git status -sb                короткий статус с веткой
    git log --oneline --graph -20
    git diff --staged             что именно уйдёт в коммит
    git add -p                    выбирать изменения по кускам
    git commit --amend --no-edit  дописать в последний коммит
    git restore <файл>            откатить файл к последнему коммиту
    git restore --staged <файл>   убрать из индекса, правки оставить
    git switch -c <ветка>         создать и перейти
    git switch -                  вернуться на предыдущую ветку

`git add -p` — главное, что стоит освоить: он заставляет посмотреть на свои
изменения перед коммитом, и коммиты сами собой становятся осмысленными.
