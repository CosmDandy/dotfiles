# tmux: команды и работа без мышки

Методичка по своему конфигу (`tools/tmux/.tmux.conf`), а не по tmux вообще.
Всё ниже — то, что реально забиндено.

Мышь **выключена** по умолчанию (`set -g mouse off`). Это осознанно: включённая
мышь ломает выделение текста средствами терминала. Разово включить —
`prefix m`, в статусбаре появится сообщение.

## Префикс

`Ctrl-s`, не стандартный `Ctrl-b`. Дальше в тексте `prefix` = `Ctrl-s`.

Когда префикс нажат, имя сессии в статусбаре справа краснеет — удобный
индикатор, что tmux ждёт вторую клавишу.

## Панели

| Клавиши | Что делает |
|---|---|
| `prefix \|` | разделить вертикально (новая панель справа) |
| `prefix -` | разделить горизонтально (новая панель снизу) |
| `Ctrl-y` `Ctrl-h` `Ctrl-a` `Ctrl-e` | перейти влево / вниз / вверх / вправо |
| `prefix y/h/a/e` | то же, если `Ctrl`-версия занята программой в панели |

Новая панель открывается **в том же каталоге**, что и текущая — это
`-c "#{pane_current_path}"` в конфиге, не поведение tmux по умолчанию.

Навигация `y/h/a/e` вместо `hjkl` — под раскладку Graphite, где эти клавиши
лежат на домашнем ряду.

Про `Ctrl-`версии важное: они работают **сквозь** Neovim. Конфиг определяет,
запущен ли в панели vim/nvim/fzf (`is_vim`), и если да — отдаёт нажатие ему,
иначе переключает панель tmux. Поэтому одни и те же четыре клавиши двигают
курсор и по сплитам nvim, и по панелям tmux, и думать о границе не нужно.

## Окна и сессии

| Клавиши | Что делает |
|---|---|
| `prefix c` | новое окно |
| `prefix 1`…`9` | перейти к окну по номеру (нумерация с 1, не с 0) |
| `prefix s` | меню сессий — список с горячими клавишами, текущая помечена точкой |
| `prefix S` | новая сессия, спросит имя |
| `prefix L` | переключиться на предыдущую сессию |
| `prefix X` | убить сессию, с подтверждением |
| `prefix d` | отсоединиться (сессия продолжает работать) |
| `prefix r` | перечитать конфиг |

Сессия при создании **автоматически называется по каталогу**, из которого
запущена, а окна переименовываются по текущему каталогу панели. Руками имена
задавать не нужно.

`detach-on-destroy off`: когда убиваешь последнюю сессию, tmux не выкидывает
в шелл, а переключает на другую живую.

# Работа без мышки: copy-mode

Это ответ на «листать и искать без мышки». Всё происходит в **copy-mode** —
режиме, где панель превращается в буфер с vim-навигацией.

**Вход:** `prefix [`
**Выход:** `Escape`

История огромная — `history-limit 1000000`, то есть миллион строк назад.

## Перемещение

Внутри copy-mode работают обычные vi-движения, потому что стоит
`mode-keys vi`:

| Клавиши | Что делает |
|---|---|
| `k` / `j` | строка вверх / вниз |
| `Ctrl-u` / `Ctrl-d` | полэкрана вверх / вниз |
| `Ctrl-b` / `Ctrl-f` | страница вверх / вниз |
| `g` / `G` | в начало / конец истории |
| `0` / `$` | начало / конец строки |
| `w` / `b` | слово вперёд / назад |
| `{` / `}` | абзац вверх / вниз |

**Шероховатость.** Движения здесь — стандартные vi-шные `hjkl`, они НЕ
переопределены под Graphite, в отличие от навигации по панелям. То есть в
copy-mode пальцы работают иначе, чем везде. Если будет мешать — лечится
четырьмя строками `bind -T copy-mode-vi`.

## Поиск

| Клавиши | Что делает |
|---|---|
| `/` | искать вперёд (вниз по истории) |
| `?` | искать назад (вверх — обычно нужно именно это) |
| `n` / `N` | следующее / предыдущее совпадение |

Типичный сценарий: команда что-то напечатала и уехало вверх. `prefix [`, затем
`?ошибка`, `Enter` — и ты на нужной строке.

## Копирование

| Клавиши | Что делает |
|---|---|
| `v` | начать выделение |
| `Ctrl-v` | прямоугольное выделение |
| `y` | скопировать, **оставшись** в copy-mode |
| `Enter` | скопировать и выйти |
| `Escape` | отменить |

`y` намеренно не выходит из режима: часто нужно скопировать несколько кусков
подряд, и каждый раз входить заново — лишние движения.

Скопированное уезжает в системный буфер (`set-clipboard on`), то есть
вставляется куда угодно, включая другие приложения и другую машину по ssh.

## Быстрое копирование без выделения: tmux-thumbs

`prefix f` — на экране подсвечиваются все «интересные» строки: пути, хеши,
URL, IP, номера. Рядом с каждой появляется буква-метка оранжевым. Жмёшь букву —
строка в буфере.

Заглавная буква вместо строчной — скопировать **и сразу вставить**.

Это самый быстрый способ забрать путь из вывода команды: не нужно ни входить в
copy-mode, ни целиться выделением.

## Вся история в Neovim

`prefix P` — сбрасывает весь скроллбек панели в файл и открывает его в nvim в
новом окне.

Нужно, когда вывода много и хочется настоящего поиска: регулярки, несколько
буферов, `:g/шаблон/`, фолды. В copy-mode это делать неудобно.

## Малоизвестное, но полезное

Всё это родные возможности tmux — в конфиге не переопределены, работают как есть.

| Клавиши | Что делает |
|---|---|
| `prefix t` | **часы на весь экран** — крупные цифры поверх панели, выход любой клавишей |
| `prefix z` | развернуть панель на всё окно и обратно; сплиты сохраняются |
| `prefix w` | список всех окон всех сессий деревом, с превью |
| `prefix !` | вынести текущую панель в отдельное окно |
| `prefix {` / `prefix }` | поменять панель местами с предыдущей / следующей |
| `prefix Space` | перебрать готовые схемы расположения панелей |
| `prefix q` | показать номера панелей; нажать цифру — прыжок |
| `prefix :` | командная строка tmux |
| `prefix ?` | список **всех** текущих биндингов — быстрее, чем лезть в конфиг |

`prefix z` — то, что чаще всего не знают и больше всего экономит: не нужно
закрывать сплиты, чтобы временно поработать в одной панели во весь экран.

`prefix q` с цифрой быстрее навигации стрелками, когда панелей больше трёх.

Про `prefix :` — там доступны все команды, включая те, что не забиндены:
`resize-pane -D 10`, `join-pane -t :2`, `swap-window -t 1`, `move-window -r`
(перенумеровать окна).

## Первый запуск

Единственный плагин — tmux-thumbs — приезжает из nix-профиля (не tpm):
конфиг подключает его условно, если файл на месте. Если `prefix f` не
работает — проверить, что home-manager switch прошёл.

## Что стоит знать про звук

`monitor-bell on`: если в фоновой панели что-то звякнуло, на macOS проиграется
системный звук Glass. Активность (просто вывод текста) при этом НЕ отслеживается —
иначе уведомления сыпались бы постоянно.
