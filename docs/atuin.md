# Atuin: быстрее находить в истории

По своему конфигу (`tools/atuin/config.toml`). Всё проверено на установленной
версии 18.17.

Синхронизация сейчас не работает — сервера `atuin.cosmdandy.ru` нет. На поиск
это не влияет: история локальная, всё ниже работает.

## Вход в поиск

    Ctrl-R      поиск по истории
    ↑           поиск с подстановкой уже набранного

Обе клавиши биндит `atuin init zsh`, никаких своих переопределений нет.

## Что уже настроено и что из этого следует

| Настройка | Значение | Практический смысл |
|---|---|---|
| `search_mode` | `fuzzy` | буквы ищутся вразбивку: `gcm` найдёт `git commit -m` |
| `filter_mode` | `host` | по умолчанию видно историю **только этой машины** |
| `enter_accept` | `true` | Enter **сразу выполняет** команду, не вставляет в строку |
| `ctrl_n_shortcuts` | `true` | `Ctrl-1`…`Ctrl-9` — выбрать N-й результат не стрелками |
| `inline_height` | `0` | поиск открывается на весь экран |
| `secrets_filter` | `true` | строки, похожие на токены, в историю не пишутся |
| `ai.enabled` | `false` | подсказок от модели нет |

Два пункта отсюда экономят больше всего времени.

**`Ctrl-1`…`Ctrl-9`.** Нужная команда почти всегда во втором-третьем результате.
Вместо трёх нажатий стрелки — одно.

**`enter_accept = true` — и связанная ловушка.** Enter выполняет немедленно.
Если команду надо **поправить** перед запуском, жать нужно `Tab`: он кладёт
строку в буфер и оставляет курсор. Это главный источник случайных запусков
не той команды.

## Внутри окна поиска

    Tab              вставить в строку без выполнения
    Enter            выполнить сразу
    Ctrl-1..9        выбрать N-й результат
    Esc / Ctrl-C     выйти
    Ctrl-D           удалить запись из истории
    Ctrl-O           переключить режим поиска

В конфиге стоит `show_help = false`, поэтому подсказка по клавишам в окне не
выводится. Если что-то из раскладки забудется — временно поставить `true` в
`tools/atuin/config.toml` и посмотреть строку подсказки, она всегда актуальна
для установленной версии.

### Режимы фильтра — самое недооценённое

По умолчанию `filter_mode = "host"`, то есть видна история **только этой
машины**. Всего режимов шесть:

- `host` — текущая машина (стоит сейчас)
- `global` — вся история со всех машин
- `session` — только текущая сессия терминала
- `directory` — только команды, запускавшиеся **в этом каталоге**
- `workspace` — в пределах текущего git-репозитория

`directory` и `workspace` отвечают на вопрос «что я запускал в этом проекте» —
именно то, ради чего обычно листают историю руками. Из командной строки они
доступны сразу:

    atuin search --filter-mode directory terraform
    atuin search --filter-mode workspace make

## Поиск из командной строки, без UI

Здесь atuin сильнее любого `history | grep`, и это работает в скриптах.

    atuin search docker                    просто найти
    atuin search --cwd . terraform         только запуски в текущем каталоге
    atuin search --exit 1 make             только те, что упали
    atuin search --exclude-exit 0 ansible  всё, кроме успешных
    atuin search --after "yesterday" kubectl
    atuin search --before "2026-07-01" --limit 20 psql
    atuin search --cmd-only --limit 5 rsync   только текст команд, без метаданных

`--cmd-only` удобен для подстановки:

    $(atuin search --cmd-only --limit 1 terraform)

Опасное, но иногда нужное: `--delete` удаляет всё, что попадает под запрос —
например, если в историю всё-таки уехал секрет.

## Статистика

    atuin stats              за всё время
    atuin stats last week
    atuin stats -c 20        топ-20 вместо топ-10
    atuin stats -n 2         пары команд подряд, а не одиночные

`-n 2` показывает связки вроде `git add` → `git commit` — из них видно, что
пора завернуть в алиас или функцию.

Учти: из статистики исключены `cd`, `ls`, `vi`, `c`, `e`, `ds`, `dpd`, `v` —
это `stats.ignored_commands` в конфиге, иначе топ состоял бы из них.

## Если понадобится синхронизация

Адрес сервера уже прописан, `auto_sync = true`, интервал 30 минут. Понадобится
только `atuin login`. Важно помнить: история шифруется end-to-end, сервер хранит
шифротекст, и ключ `~/.local/share/atuin/key` невосстановим — при входе на новой
машине его нужно передать явно (`atuin key` на старой). Ключ попадает в слой
секретов `automation/backup/backup.sh`.
